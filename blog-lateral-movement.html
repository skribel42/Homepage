Lateral Movement Without an RMM Agent

Popping shells and lateral movement is so much more fun in real life. How I used PowerShell to reach an unmanaged server and get it under management — no truck roll required.

powershell windows sysadmin offensive security

Every IT pro eventually runs into a machine that fell through the cracks. No RMM agent, no remote access, sitting at a remote site that nobody wants to drive to. In a managed services environment, that machine is a liability — you can't patch it, can't monitor it, can't touch it without physically being there.

This is one of those stories. And it's also a great example of why learning offensive security techniques makes you a better sysadmin.

// The Situation

A server at a remote client site wasn't showing up in our RMM platform. No agent, no access. The site was hours away — a truck roll wasn't happening for something like this. But I did have credentials and access to other endpoints on the same network. That's all I needed.

The technique is essentially lateral movement — pivoting from a machine you control to one you don't, using tools and protocols that already exist on the network. No exploits, no malware. Just PowerShell doing what PowerShell does.

// Step 1: Get a Shell

From an endpoint I already had access to on the same network, I tried to establish a remote PowerShell session using WinRM:

Enter-PSSession -ComputerName TARGET_IP -Credential DOMAIN\administrator

If you hit a TrustedHosts error connecting via IP address, add the target to your trusted hosts list first:

Set-Item WSMan:\localhost\Client\TrustedHosts -Value "TARGET_IP" -Force

Then retry the session. When it connected, I did what any self-respecting hacker does first:

whoami

Confirmed. I had a shell on the target machine.

// Step 2: Enable RDP

A PowerShell session works, but it's not exactly comfortable for installing software. I needed a full desktop. Two commands to enable RDP remotely:

# Enable RDP
Set-ItemProperty -Path 'HKLM:\System\CurrentControlSet\Control\Terminal Server' `
    -Name "fDenyTSConnections" -Value 0

# Open the firewall
Enable-NetFirewallRule -DisplayGroup "Remote Desktop"

Then from my local machine, RDP straight in:

mstsc /v:TARGET_IP

Full desktop. Server with IE Enhanced Security locked down, no browser worth using — but that's fine, we don't need one.

// Step 3: Drop the Payload

I needed to get the RMM agent installer onto the machine without a usable browser. PowerShell handles that too:

Invoke-WebRequest -Uri "YOUR_RMM_INSTALLER_URL" -OutFile "C:\Windows\Temp\rmm-agent.exe"
Start-Process "C:\Windows\Temp\rmm-agent.exe"

Agent installed, machine phoned home to the RMM platform, endpoint under management. No truck roll. Total time: under 20 minutes.

// Why This Matters

Everything I did here — PSSession, registry edits, firewall rules, remote file downloads — is exactly what an attacker does when they're moving through a network. The difference is authorization and intent.

Understanding how lateral movement works isn't just useful for offensive security. It makes you a better defender because you understand what to look for, and a better sysadmin because you have more tools at your disposal when things go wrong.

The next time someone tells you offensive security knowledge is only for red teamers, tell them about the server that didn't need a truck roll.
