Homelab HA on a Budget: NFS + Proxmox + a USB Drive

How I got shared storage and Pi-hole high availability running across a 3-node mini PC cluster for basically nothing.

proxmox homelab nfs linux

Everyone's homelab journey hits the same wall eventually — you get a cluster running, you want high availability, and then you realize HA requires shared storage. And shared storage usually means a NAS, a SAN, or Ceph. All of which cost either money or a machine you don't have.

Mine hit that wall when I decided I wanted Pi-hole to automatically fail over between nodes if one went down. I had a 3-node Proxmox cluster running on mini PCs, no extra hardware, and a 2TB SATA drive sitting in a drawer. Here's how I made it work for basically nothing.

// The Setup

Three mini PCs running Proxmox VE, all clustered up, all on the same management network. Each node has a single drive — the OS and any VMs living on the same disk. No spare drives, no NAS, no Ceph. Pi-hole running as an LXC container on one of the nodes, DNS working fine, no redundancy whatsoever.

The problem with HA in this situation is straightforward: if the node running Pi-hole dies, Proxmox can detect the failure and try to restart the container on another node — but the container's disk is sitting on the dead node's local storage. The other nodes can't see it. Failover fails.

The fix is shared storage. All nodes need to be able to see the same disk so any of them can pick up where the other left off.

// The Hardware

I had a 2TB 3.5" SATA drive and picked up a powered USB SATA dock. The "powered" part matters — unpowered docks are the most common cause of intermittent disconnects, and a storage device that occasionally drops off the network is worse than no storage device at all.

Is USB ideal for shared storage? No. Is it fine for a homelab running Pi-hole? Absolutely. The write load on a DNS container is minimal and USB 3.0 throughput is more than enough.

// Step 1: Format the Drive

Plug the drive into one of your nodes and find it:

lsblk -o NAME,SIZE,TYPE,TRAN,MODEL,MOUNTPOINT

Look for the USB device — it'll show usb in the TRAN column. Mine showed up as /dev/sdb. Wipe any old partitions and format it:

wipefs -a /dev/sdb
mkfs.ext4 /dev/sdb

// Step 2: Mount It and Set Up NFS

Create a mount point, mount the drive, add it to fstab so it survives reboots, then install and configure the NFS server:

mkdir /mnt/shared
mount /dev/sdb /mnt/shared
echo "/dev/sdb /mnt/shared ext4 defaults 0 2" >> /etc/fstab
apt install -y nfs-kernel-server
echo "/mnt/shared 10.10.10.0/24(rw,sync,no_subtree_check,no_root_squash)" >> /etc/exports
exportfs -a
systemctl enable --now nfs-kernel-server

Replace the subnet with whatever your Proxmox nodes are on. Verify the share is live:

exportfs -v

// Step 3: Mount on the Other Nodes

On every other node in the cluster, mount the NFS share and add it to fstab:

mkdir /mnt/shared
mount 10.10.10.X:/mnt/shared /mnt/shared
echo "10.10.10.X:/mnt/shared /mnt/shared nfs defaults 0 0" >> /etc/fstab
systemctl daemon-reload

Replace 10.10.10.X with the IP of the node hosting the drive.

// Step 4: Register as Proxmox Shared Storage

Tell Proxmox about it so it shows up as a storage option across the whole cluster. Run this on any one node:

pvesm add nfs shared --server 10.10.10.X --export /mnt/shared --content images,rootdir --nodes Proxmox1,Proxmox2,Proxmox3

Verify it shows up and is active:

pvesm status | grep shared

// Step 5: Move Pi-hole to Shared Storage

Shut down the container first, then move its disk to the shared storage:

pct stop 901
pct move-volume 901 rootfs --storage shared

This copies the container's root filesystem to the NFS share. Now any node can access it.

// Step 6: Enable HA

In the Proxmox GUI: Datacenter → HA → Add, select the container, set state to started. That's it. Start the container back up and you're done.

Now if the node running Pi-hole goes offline, Proxmox detects the failure, pulls the container's disk from the NFS share, and starts it on a healthy node automatically. Your DNS barely hiccups.

// The Caveat

The node hosting the USB drive is now a single point of failure for shared storage. If that node dies, the NFS share goes with it and HA can't do its job. For a homelab this is an acceptable tradeoff. For production you'd want Ceph or a proper NAS with redundancy. But for getting your feet wet with Proxmox HA without spending anything? This gets the job done.
